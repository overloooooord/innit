# InVision U — AI Candidate Scoring System

> **Hackathon:** Decentrathon 5.0 · **Track:** AI inDrive · **Deadline:** April 5, 2026

InVision U — образовательная программа от inDrive для молодых людей **16–22 лет из Казахстана**. Полный грант: обучение, проживание, менторы. Отбор по потенциалу, а не по оценкам.

---

## Философия проекта

Большинство систем отбора оценивают кандидата как **снимок** — что у него есть прямо сейчас. Это принципиально несправедливо: 17-летний из Астаны с доступом к репетиторам и олимпиадам стартует с другой точки, чем 17-летний из посёлка в Туркестанской области.

**Наша система оценивает не то, где кандидат находится, а то, как далеко он прошёл от своей стартовой точки и как быстро растёт.**

Кандидат с GPA 4.2 из Актау, который самостоятельно основал эко-проект и организовал 40 человек, получает более высокий скор, чем кандидат с GPA 4.8 из Алматы, который только ходил в кружки.

> **Ключевой принцип:** AI не принимает финальных решений — он предоставляет прозрачные рекомендации. Окончательное слово всегда за приёмной комиссией (human-in-the-loop).

## Архитектура системы

```
[PROACTIVE SEARCH]
Учитель номинирует ученика
        │
        ▼
nomination_analyzer.py (NLP-модуль)
  Извлекает лидерские сигналы из текста учителя
  → nomination_pre_score (только для приоритизации приглашений)
  → Отправляет персональное приглашение кандидату
        │
        ▼
[CANDIDATE JOURNEY]
Кандидат → Telegram Bot (state machine, без AI)
  1. Экран согласия (явное согласие на данные)
  2. Структурированные вопросы → JSON-поля кандидата
  3. Эссе с якорем на эпизодическую память (анти-LLM)
  4. Leadership Fingerprint сценарии (5 ветвящихся ситуаций, таймер 20 сек)
        │
        ▼
[FEATURE ENGINEERING]
  структурные поля       → 18 фичей
  leadership fingerprint → 5 features from SLPI (cornell uni)
  NLP-модуль             → 6 features, also SLPI (cornell)
        │
        ▼
[MODEL]
XGBoost + 5-fold cross-validation
  вход: 18 структурных фич + 500-мерный fingerprint-вектор
  предсказание: shortlist / maybe / reject
        │
        ▼
[EXPLAINABILITY]
SHAP values
  структурные фичи: топ-5 позитивных факторов, топ-3 слабости
  fingerprint: 5 агрегированных SLPI-кластеров для людей
        │
        ▼
[FAIRNESS AUDIT]
Независимая система — проверяет, дискриминирует ли модель
по городу, типу школы, региону (НЕ используется в скоринге)
        │
        ▼
FastAPI Backend → React Dashboard
  Комиссия видит карточку кандидата → нажимает Shortlist / Review / Reject
  Человек принимает финальное решение
```

## Путь кандидата шаг за шагом

### Шаг 1 — Проактивный поиск

Учитель из Шымкента заполняет форму номинации на Асель → `nomination_analyzer.py` выявляет: «организовала конференцию», «убедила администрацию», «не сдалась после первого отказа» → `nomination_pre_score: 0.74` → бот отправляет персональное сообщение: «Привет, Асель, твой учитель рассказал нам о твоей конференции...»

> **Важно:** `nomination_pre_score` **никогда** не входит в финальную модель скоринга. Номинированный и самоприменившийся кандидат проходят идентичный пайплайн.

### Шаг 2 — Telegram Bot (7 блоков)

Бот реализован как **чистый state machine** без LLM — мгновенный отклик, никакого риска джейлбрейка, одинаковый опыт для всех кандидатов.

| Блок | Что происходит | Что собирается |
|------|---------------|----------------|
| **1. Приветствие** | Снятие тревоги, объяснение формата | `funnel_stage`, `start_timestamp` |
| **2. Согласие** | Явный GDPR-экран, юридически обязательный | `consent_given`, `consent_timestamp` |
| **3. Личные данные** | Имя, возраст, город | `personal.name/age/city` |
| **4. Образование** | GPA, олимпиады (до 10), курсы (до 10) | `education.*` |
| **5. Опыт** | Проекты и активности (до 10), роли, трудности | `experience.projects[*]` |
| **6. Эссе** | 150–300 слов, якорный вопрос на эпизодическую память | `essay.text`, `essay.word_count` |
| **7. Сценарии** | 5 ветвящихся ситуаций с таймером 20 сек | `bot_metadata.scenario_choices` |

#### Блок 6 — Анти-LLM якорь в эссе

Вместо открытого вопроса («расскажи о себе») кандидата просят вспомнить **один конкретный момент** — день, ситуацию, одно решение. LLM без персонального контекста генерирует обобщённый, абстрактный текст («я всегда стараюсь», «это научило меня»). Настоящий эпизодический ответ содержит конкретные детали: дату, место, имена, специфические действия, эмоциональный переход. NLP-модуль анализирует конкретность как сигнал аутентичности.

### Шаг 3 — Feature Extraction

Пример для Асель (Актау, основатель эко-проекта, 40 волонтёров):

```
f_gpa = 4.2
f_project_count = 3

```

## Killer Feature: Leadership Fingerprint Engine

### Проблема

Все существующие системы просят кандидатов «расскажите о себе». Два фатальных изъяна:
1. ChatGPT генерирует идеальные ответы на стандартные открытые вопросы за секунды
2. Артикулированные кандидаты побеждают; застенчивые кандидаты с реальным потенциалом проигрывают

### Решение: ветвящийся сценарный движок

Кандидат проходит **5 micro-ситуаций с таймером 20 секунд** прямо в Telegram-боте. Каждый последующий вопрос зависит от предыдущего ответа — предгенерация невозможна.

Всего в банке: **25 вопросов × 4 варианта = 100 ответов** (5 сценариев × 5 вопросов в каждом). Каждый кандидат видит **10 вопросов** (5 входных + 5 ветвей по своим выборам).

| Сценарий | SLPI-фокус |
|----------|-----------|
| 1 — Организационный кризис | Enable Others to Act · Model the Way |
| 2 — Конфликт в команде | Encourage the Heart · Enable Others to Act |
| 3 — Идея против системы | Challenge the Process · Inspire a Shared Vision |
| 4 — Поддержка человека | Encourage the Heart · Enable Others to Act |
| 5 — Ресурсный конфликт | Model the Way · Inspire a Shared Vision |

**Нет правильного ответа.** Паттерн выборов по всем шагам формирует fingerprint-вектор.

### Пример сценарного дерева (Сценарий 1)

```
⏱ 20 секунд
Ты отвечаешь за школьное мероприятие. За 3 дня до события
главный организатор выбыл — заболел. Что делаешь?

[A] Беру всё на себя — справлюсь
[B] Собираю команду и распределяю задачи
[C] Срочно ищу замену организатору
[D] Сообщаю куратору / классному руководителю

→ При [A]: «К вечеру понимаешь, что не успеваешь. Задач слишком много. Что дальше?»
→ При [B]: «Один человек в команде не справляется. До мероприятия 2 дня. Что делаешь?»
→ При [C]: «Новый организатор просит больше бюджета — иначе не берётся. Что делаешь?»
→ При [D]: «Куратор отвечает: это ваша ответственность. Как реагируешь?»
```

### Почему нельзя подготовиться заранее

| Атака | Защита |
|-------|--------|
| Сгенерировать эссе через ChatGPT | Якорь на эпизодическую память требует конкретных личных деталей |
| Сгенерировать сценарии через ChatGPT | Каждый вопрос зависит от предыдущего — дерево неизвестно заранее |
| Скопировать с телефона | Таймер 20 сек — бот автоматически переходит дальше |
| Открытый текст для AI | Только multiple choice — нечего генерировать |
| Запомнить «правильные» ответы | Правильных ответов нет — оценивается паттерн, не отдельный выбор |

### 500-мерный вектор
5 сценариев × 4 варианта × 25 поведенческих весов = 500 чисел

Каждый выбор кандидата (например, сценарий 2, вариант B) вносит
взвешенный вклад в 25 элементов вектора — по каждому из 5 SLPI-измерений
свой набор весов. Итого 500 float-значений.

# scenario_engine.py
def compute_fingerprint(choice_path: list[str]) -> dict:
    """
    choice_path: list длиной 5 — по одному выбору на каждый сценарий
    Например: ["B", "C", "A", "B", "D"]

    Возвращает:
      fingerprint_vector: list[float], len=500  → входит в ML-модель
      fingerprint_display: dict, 5 SLPI-значений → дашборд + SHAP
    """
    fingerprint_vector = encode_choices_to_vector(choice_path)
    fingerprint_display = aggregate_to_slpi(fingerprint_vector)
    return {
        "fingerprint_vector": fingerprint_vector,
        "fingerprint_display": fingerprint_display
    }

XGBoost получает полный 500-мерный вектор (нелинейные паттерны).
Комиссия видит 5 значений (понятно людям без объяснений).

Пример результата (путь B→C→A→B→D):
fingerprint_display:
  model_the_way:     0.52
  inspire_vision:    0.38
  challenge_process: 0.44
  enable_others:     0.81
  encourage_heart:   0.74

Каждое из 5 SLPI-измерений присутствует минимум в **3 сценариях** — надёжное покрытие даже при timeout на отдельных шагах.

## Feature Engineering

Всего: **18 структурных фич + 500-мерный fingerprint-вектор**

### Education Features
- GPA (нормализованный из raw-строки)
- Количество олимпиад
- Максимальный уровень олимпиады (международный / республиканский / областной / школьный)
- Разнообразие предметов олимпиад
- Наличие призового места
- Количество курсов
- Коэффициент завершения курсов

### Experience Features
- Количество проектов
- Founder ratio (основатель+соосноватетель / всего проектов)
- Максимальная роль (ранг роли по всем проектам)
- Project diversity (уникальные типы проектов)
- Max team size (с log-масштабированием)

### Trajectory Features (считаются по блокам 4+5 вместе)
- Role progression — участник → основатель во времени
- Scope progression — рост размеров команды и уровней олимпиад по годам
- Skill diversity growth — новые типы навыков в год
- Activity years span — диапазон активных лет
- Persistence signal — признал неудачу И продолжил после
- Failure acknowledgment ratio — доля проектов с описанием трудностей

### NLP Features (отдельный модуль)
- Leadership Framework Based on Kouzes & Posner S-LPI:
- model_the_way — оценивает, насколько кандидат демонстрирует личный пример и следует своим ценностям на практике.
- inspire_shared_vision — оценивает, умеет ли кандидат формулировать видение и вовлекать других в общую цель.
- challenge_the_process — оценивает готовность кандидата предлагать новые подходы и менять существующие процессы.
- enable_others_to_act — оценивает, насколько кандидат помогает другим развиваться, брать ответственность и достигать результатов.
- encourage_the_heart — оценивает, поддерживает ли кандидат других, признаёт их вклад и укрепляет командный дух.

### Что намеренно исключено из скоринга

| Поле | Причина |
|------|---------|
| `city` | Демографический прокси для социоэкономического статуса |
| `school_type` | Прокси для доступа к ресурсам |
| `has_mentor` | Прокси для социального капитала |
| `nomination_pre_score` | Создаёт неравенство между номинированными и самоприменившимися |
| `ai_detection_flag` | Показывается комиссии в карточке — никогда не в формуле |
| Время печати, паузы | Ненадёжный сигнал, bias против уверенных людей |

Все исключённые поля **видны комиссии в карточке**, но **никогда не входят в скоринговую формулу**.

## Почему Telegram Bot, а не Google Form

Google Form — «мёртвая» анкета. Красиво пишущие побеждают; застенчивые проигрывают. ChatGPT генерирует идеальные ответы за секунды.

### Почему НЕ LLM-бот

| Проблема | LLM-бот | State Machine Bot |
|----------|---------|-------------------|
| Скорость | 2–5 сек на вызов API | Мгновенно |
| Риск джейлбрейка | Prompt injection может сломать извлечение | Невозможно — нет LLM |
| Галлюцинации | LLM может додумать данные | Сохраняет raw-текст как есть |
| Стоимость | 40 000+ вызовов API на 1000 кандидатов | Ноль |
| Справедливость | Разный опыт у разных кандидатов | Все получают идентичный опыт |

**Интеллект — в ML-пайплайне ПОСЛЕ сбора данных, а не во время него.**

## Известные ограничения и митигации

**1. Синтетические обучающие данные** — риск: модель учит паттерны автора, не реальные. Митигация: inter-annotator agreement с 3 независимыми разметчиками + Cohen's kappa. Adversarial test set из 15 trick-профилей.

**2. Telegram privacy risk** — данные проходят через серверы Telegram. Митигация: явный экран согласия до сбора данных. Рекомендация для продакшна: переезд на self-hosted бота.

**3. Fingerprint сценарии без реальной валидации** — маппинг сценарий→fingerprint основан на теоретической модели. Митигация: adversarial профили включают «хамелеонов». Ablation study покажет вклад fingerprint. Цель для фазы 2.

**4. Косвенный bias через proxy-фичи** — `courses_count` может коррелировать с `city`. Митигация: fairness audit явно проверяет proxy-корреляции и флагирует при обнаружении.

python feature_extractor.py ../data/sample_candidate.json
