## 🎨 Темы (тёмная / светлая)

**Как работает:**

1. В CSS определены **две группы переменных**:
   ```css
   [data-theme="dark"] {
     --clr-bg: #05060f;     /* тёмный фон */
     --clr-text: #e2e8f0;   /* светлый текст */
   }
   [data-theme="light"] {
     --clr-bg: #f0f2f5;     /* светлый фон */
     --clr-text: #1a1a2e;   /* тёмный текст */
   }
   ```

2. **Все** элементы используют эти переменные:
   ```css
   body { background: var(--clr-bg); color: var(--clr-text); }
   ```

3. При клике на 🌙/☀️ JavaScript меняет атрибут:
   ```js
   document.documentElement.setAttribute('data-theme', 'light');
   ```

4. CSS **мгновенно** перерисовывает все цвета (через `transition: 0.3s`).

5. Выбор сохраняется в `localStorage` → не сбрасывается при перезагрузке.

## 🌐 Смена языка (i18n)

**Как работает:**

1. В HTML у каждого текста стоит атрибут `data-i18n`:
   ```html
   <h2 data-i18n="reg_title">Заявка</h2>
   ```

2. В `i18n.js` хранится словарь переводов:
   ```js
   TRANSLATIONS = {
     ru: { reg_title: "Заявка" },
     kz: { reg_title: "Өтінім" },
     en: { reg_title: "Application" },
   }
   ```

3. При клике на кнопку языка (RU / ҚЗ / EN):
   ```js
   setLanguage('kz');
   // → Находит ВСЕ элементы с data-i18n
   // → Заменяет текст на перевод
   // → Сохраняет выбор в localStorage
   ```

## 📝 MBTI тест

**Как работает:**

1. 40 вопросов хранятся в массиве `MBTI_QUESTIONS` (в `app.js`)
2. Показываем **по одному** вопросу — пошаговый интерфейс
3. state.mbtiIndex = номер текущего вопроса (0–39)
4. state.mbtiAnswers = `{ q1: "A", q2: "B", ... }`
5. Прогресс-бар: `width = (index + 1) / 40 * 100%`
6. На 40-й → отправляем POST `/api/tests/mbti/`
7. Backend рассчитывает тип (INTJ, ENFP и т.д.)

## ⏱ Языковой тест

**Как работает:**

1. 20 вопросов с 4 вариантами (A/B/C/D)
2. **Таймер**: 10 минут (600 секунд), каждую секунду -1
   - Зелёный → Жёлтый (< 3 мин) → Красный мигающий (< 1 мин)
   - При 0 → автоматическая отправка
3. **Контроль вкладки**: `document.visibilitychange`
   - 1-й выход → предупреждение (toast)
   - 2-й выход → блокировка + автоотправка
4. Результат: `{ score, max_score, time_spent_seconds, violation_count }`

## 📡 API

**Endpoints бэкенда:**

| Метод | URL                        | Что делает                     |
|:------|:---------------------------|:-------------------------------|
| GET   | `/api/cities/`             | Список городов и языков        |
| GET   | `/api/cities/<город>/regions/` | Регион для города          |
| POST  | `/api/applications/`       | Создать заявку                 |
| POST  | `/api/tests/mbti/`         | Отправить MBTI результаты      |
| POST  | `/api/tests/language/`     | Отправить результат язык.теста |

**Пример вызова (из api.js):**
```js
const result = await API.submitApplication({
  name: "Айгерим",
  city: "Алматы",
  region: "город Алматы",
  languages: ["Казахский", "Русский"],
  telegram_username: "@aigerim",
  hobbies: "Чтение",
  sport: "Плавание",
});
// result = { id: 42, name: "Айгерим", ... }
```

## ⚠️ Обработка ошибок

1. **Нет интернета** → `TypeError: Failed to fetch` → toast "Нет связи с сервером"
2. **Backend вернул ошибку** → статус 400/500 → toast с текстом ошибки
3. **Валидация** → backend проверяет город, регион, языки, telegram → ошибки
4. **Fallback** → если backend недоступен, города загружаются из встроенного списка
